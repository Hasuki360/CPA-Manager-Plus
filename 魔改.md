# CPA Manager Plus 魔改记录

## 目标

- 配置页的“额度处理”增加 Antigravity 版开关。
- Antigravity 额度用完时，也能像 Codex 一样临时停用对应登录文件，并在恢复时间后自动启用。
- Manager Server 管理员访问密码允许简单密码；当前后端只要求非空，本次补测试防止以后被改回强制 32 位。
- 请求监控实时表格增加列显示开关，按钮放在“失败”筛选旁边。
- 请求监控实时表格“用量”列在中文界面保留 `I/O/R/C` 标签，但数字单位从 `K/M` 改为 `万/亿`。

## 后端改动

- `apps/manager-server/internal/config/config.go`
  - 新增配置文件字段：`antigravityQuotaCooldownEnabled`
  - 新增环境变量：`USAGE_ANTIGRAVITY_QUOTA_COOLDOWN_ENABLED`
  - 新增环境锁定标记：`AntigravityQuotaCooldownEnvSet`

- `apps/manager-server/internal/model/automation_settings.go`
  - 新增 DB/UI 覆盖项：`antigravityQuotaCooldownEnabled`

- `apps/manager-server/internal/service/automation/service.go`
  - `GET /usage-service/account-processing-policy` 返回 `antigravityQuotaCooldown`
  - `PATCH /usage-service/account-processing-policy` 支持 `antigravityQuotaCooldownEnabled`
  - 运行时设置新增 `AntigravityQuotaCooldownEnabled`

- `apps/manager-server/internal/worker/automation_runtime.go`
  - Codex 与 Antigravity 的额度冷却开关独立生效。
  - 只有对应 provider 的开关打开时，才把事件交给 quota cooldown worker。

- `apps/manager-server/internal/worker/rate_limit_auto_disable.go`
  - 原来只处理 `provider=codex` 的 `429 usage_limit_reached`。
  - 现在同时支持 `provider=antigravity`。
  - 仍然要求能解析出明确恢复时间，避免误停用。

- `apps/manager-server/internal/security/security_test.go`
  - 新增简单管理员密码测试，例如 `123456` 可以创建凭据并验证。

## 使用方式

### 本地 Docker 启动验证

当前项目按 Docker-only 部署，不使用原生安装脚本。

当前本地目录结构：

```text
/www/234G/CPA-Manager-Plus/
├── 魔改/      # Hasuki360/CPA-Manager-Plus，实际部署源码
├── 原版/      # seakee/CPA-Manager-Plus，干净上游参照
└── 历史备份/  # 本地备份包
```

本机验证时进入魔改目录，并用临时 override 覆盖管理员密钥，避免把固定密码写入仓库：

```bash
cat >/tmp/cpamp-local-override.yml <<'EOF'
services:
  cpa-manager-plus:
    image: hasuki/cpa-manager-plus:local
    environment:
      CPA_MANAGER_ADMIN_KEY: "cpamp_local_change_me"
EOF

cd /www/234G/CPA-Manager-Plus/魔改
docker compose -p cpa-manager-plus -f docker-compose.manager.yml -f /tmp/cpamp-local-override.yml up -d --build
```

访问：

```text
http://localhost:18317/management.html
```

健康检查：

```bash
curl -fsS http://127.0.0.1:18317/health
```

### VPS Docker-only 部署

VPS 只使用 Docker 模式：拉取仓库源码，本地构建镜像并启动容器。

```bash
git clone https://github.com/Hasuki360/CPA-Manager-Plus.git
cd CPA-Manager-Plus
export CPA_MANAGER_ADMIN_KEY='换成莲纪自己的管理密钥'
docker compose -p cpa-manager-plus -f docker-compose.manager.yml up -d --build
```

后续更新：

```bash
cd CPA-Manager-Plus
git pull --ff-only origin main
export CPA_MANAGER_ADMIN_KEY='换成莲纪自己的管理密钥'
docker compose -p cpa-manager-plus -f docker-compose.manager.yml up -d --build
```

### 后期自动任务同步更新

后期可以在 VPS 上做定时任务，逻辑固定为：

1. `git fetch origin upstream`
2. 如需同步原版，先在本地服务器处理 `upstream/main` 合并到魔改分支，解决冲突并测试后推送 `origin/main`
3. VPS 只拉 `origin/main`
4. `docker compose -p cpa-manager-plus -f docker-compose.manager.yml up -d --build`
5. 健康检查 `/health`，失败则输出日志并告警

VPS 自动任务不要直接合并 `upstream/main`，避免无人值守时把上游冲突或行为变化带进生产容器。

### 简单管理员密码

后端允许简单管理员密码。部署时推荐通过环境变量或 Docker secret 设置：

```bash
export CPA_MANAGER_ADMIN_KEY='换成莲纪自己的管理密钥'
```

如果已经初始化过，需要重置管理员密钥：

```bash
cpa-manager-plus reset-admin-key --admin-key 123456
```

Docker 里可用对应容器执行，注意数据库 volume 要指向当前实例。

### Antigravity 额度冷却

可在配置页打开，也可以用环境变量锁定：

```yaml
environment:
  USAGE_ANTIGRAVITY_QUOTA_COOLDOWN_ENABLED: "true"
```

环境变量锁定后，UI 只能查看，不能修改。

## 同步上游注意事项

- 上游如果改了 `account-processing-policy` 接口，需要重点检查：
  - `automation_settings.go`
  - `service/automation/service.go`
  - `AccountProcessingPolicySection.tsx`
  - `accountProcessingPolicyViewModel.ts`

- 上游如果改了请求监控实时表格，需要重点检查：
  - `RealtimeEventsPanel.tsx`
  - `MonitoringCenterPage.tsx`
  - `MonitoringCenterPage.module.scss`

- 上游如果改了 quota cooldown worker，需要重点检查：
  - `automation_runtime.go`
  - `rate_limit_auto_disable.go`
  - 对 `provider` 的过滤是否仍保留 Antigravity。
